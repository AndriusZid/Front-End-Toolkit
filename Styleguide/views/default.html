Welcome

A style guide is a set of visual elements and guidelines for specific web project of specific organization. The goal of the style guides is to enforce consistency across the project.

For Developers

Usage

Before you start using a style guide you need to include CSS references of your project to the template's header '/styleguide/index.shtml'. Once it's done, paste your html code (snippets) inside custom tags described below.

Snippets

All snippets have to be defined inside 'snippets' folder. Each file in the folder represents separate component and will be listed in the sidebar menu as separate item. We use Grunt to automatically update sidebar menu but it can be done manually by updating '\styleguide\scripts\snippets.json' file.

To create new snippet you have to wrap your code with a custom html tag <snippet> and add title attribute to it data-snippet-title. Example:

			<snippet data-snippet-title="Name of the snippet">
				<!-- Your code goes here -->
			</snippet>
